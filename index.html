精靈驗測對照（Integration Element Status）

各頁 <head> 皆載入 assets/js/insider-page-queue.js（在 ins.js 之前），會自動依網址推送下列佇列（再送 init）：

項目 測試頁面 佇列重點
1. Page Events and Types 首頁、其餘頁之頁型事件； 結帳 僅 other → init home / category / product / cart / purchase + init
2. User Attributes 模擬登入完成頁 完整 user（範本欄位：uuid、gender、birthday、has_transacted、transaction_count、gdpr、name、surname、username、email、各 optin、language、returning、static_segment_id、age）；其餘頁為匿名 user（uuid、birthday、language、gdpr_optin）
3. Product Page Events 商品頁 product（範本欄位含 taxonomy、in_stock、locale、tags、omnibus_* 等）→ currency: USD → init
3b. Page data · Category 分類頁 匿名 user → currency: TWD → category（必填 breadcrumb 字串陣列，與文件範例相同）→ init
4. Cart Page Events 購物車 type: 'cart'（total、items[]）
4b. Checkout Page Event 結帳 僅 type: 'other' → init（與精靈文件一致；不送 user／currency）
5. Confirmation Page Events 完成頁 purchase（order_id、total、完整 items[]）→ currency: USD → init
提醒：在後台「測試 Insider Tag 整合」前，請先在此專案各頁 <head> 內貼上 Insider 提供的標籤程式碼。
